## 1. Purpose

The AppDetails screen is the premium product detail page.

It should:
- showcase one app strongly
- explain value clearly
- increase trust
- move users toward inquiry/demo/contact

This is the main conversion page after browsing.

---

## 2. Main Goal

Users should be able to:
1. understand what the app is
2. see the product visually
3. understand price/value
4. explore key features
5. compare related apps
6. request demo/contact easily

---

## 3. Screen Flow Order

Top to bottom:

1. top bar
2. hero image / gallery section
3. gallery thumbnails
4. trust strip
5. overview block
6. features block
7. delivery / ideal-for info cards
8. related apps section
9. CTA block
10. common footer
11. sticky bottom CTA

---

## 4. Section-by-Section Wireframe

## A. Top Bar

### Content
- back button
- title: `App Details`
- save button
- share button

### Purpose
- utility actions
- quick navigation
- simple product-level controls

---

## B. Hero Product Section

### Content
- large image or gallery image
- category chip
- supporting badge like `Best Seller`
- app title
- short description
- price
- quick meta:
  - `Web & Mobile`
  - `Customizable`

### Purpose
- strongest product presentation area
- immediate value communication

### Notes
- image should dominate visually
- content overlay should still stay readable
- price should stand out strongly

---

## C. Gallery Thumbnails

### Show when
- multiple gallery images exist

### Purpose
- allow product preview switching

### Notes
- active thumb should have accent border
- keep size compact and clean

---

## D. Trust Strip

### Content
- `Demo / Available`
- `Custom / Build`
- `Fast / Delivery`

### Purpose
- short confidence-building support

---

## E. Overview Block

### Content
- eyebrow: `OVERVIEW`

## F. Feature Highlights Block
