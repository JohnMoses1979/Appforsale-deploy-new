# Figma Notes — Sat Apps Marketplace

## 1. Project Context

Project name:
- Sat Apps Marketplace

Product type:
- premium app sales marketplace
- business app catalog
- custom digital solutions showcase

Main goal:
- present apps in a premium way
- help users browse products easily
- increase trust and conversions
- move users toward inquiry / demo / custom project

---

## 2. Design Direction from Figma

The visual style should feel:

- premium
- modern
- dark elegant
- glass-inspired
- conversion-friendly
- business-focused

The UI should not feel:
- generic
- too colorful
- playful
- cluttered
- template-like

---

## 3. Overall Figma Mood

Key mood references:

- dark luxury dashboard feel
- premium SaaS landing page
- polished marketplace layout
- subtle glassmorphism
- clean product presentation
- soft accent highlights

Design balance:
- rich enough to feel premium
- simple enough to stay readable
- modern enough to look product-ready

---

## 4. Layout Rules from Figma

### General layout
- mobile-first layout
- vertical stacked sections
- generous spacing between sections
- rounded cards with soft borders
- no overcrowding

### Common page structure
Each main screen should generally follow:

1. top navigation bar
2. hero / intro block
3. main content sections
4. CTA block if relevant
5. shared footer

---

## 5. Header Notes

### Top bar style
- dark glass surface
- rounded rectangle
- soft border
- compact and premium

### Header content pattern
- left action: back or brand
- center: page title
- right action: contact or compact utility action

### Notes
- avoid too many buttons
- keep visual balance
- maintain same style across all screens

---

## 6. Hero Section Notes

### Hero should communicate
- what this page is about
- the premium quality of the product/company
- a clear next step

### Hero content style
- small eyebrow text
- bold large title
- short readable subtitle
- optional supporting pills/chips
- optional primary CTA

### Hero spacing
- generous top-bottom spacing
- enough breathing room around title
- subtitle width should not feel too long

---

## 7. Card Design Notes

### Card style
- large radius
- soft light border
- glass-inspired fill
- layered gradients where needed
- premium but not overly shiny

### Card usage
Used for:
- app cards
- feature blocks
- summary blocks
- info sections
- contact cards
- CTA sections

### Card content pattern
Prefer:
- eyebrow
- title
- supporting text
- bottom action or meta row

---

## 8. App Card Notes

### App card purpose
- quick browsing
- show image, category, title, price, CTA
- feel sellable and premium

### App card structure
- image on top
- small category chip
- title
- short description
- price
- action button

### Notes
- image should feel important
- button should be clear but not oversized
- card height should remain balanced

---

## 9. Footer Notes

### Footer direction
Footer should be common across all screens.

### Footer content
- brand section
- company links
- product links
- support links
- contact information
- copyright line
- privacy / terms mini row

### Notes
- footer links should be consistent in color
- footer spacing should feel organized
- footer must wrap well on mobile
- use same footer everywhere

---

## 10. Navigation Notes

Main routes:
- Home
- Apps
- AppDetails
- About
- Contact

Navigation should feel:
- direct
- simple
- predictable

No unnecessary deep navigation layers.

## 12. Typography Notes from Figma

### Display titles
- strong
- bold
- slightly compact
- premium feeling

### Body text
- clean
- readable
- not too small
- not too dense

### Small labels
- uppercase eyebrow text
- spaced lettering
- accent color

### Notes
- titles should stand out strongly
- body text should remain calm and readable
- buttons should feel crisp and confident

---

## 13. Interaction Notes

### Press feedback
Use:
- slight opacity drop
- slight scale down

### Motion style
Use:
- soft fade-up
- staggered reveal
- subtle section motion
- no aggressive animations

### Scroll behavior
- sticky header where useful
- smooth vertical flow
- horizontal scroll only where needed
- maintain performance

---

## 14. Figma Consistency Rules

Keep consistent:
- card radius
- border color
- section spacing
- typography scale
- button style
- chip style
- footer structure
- page padding

Do not randomly change:
- colors per page
- footer style
- top bar treatment
- CTA style
- chip design

---

## 15. Content Notes

### Tone of writing
Should feel:
- premium
- clear
- trustworthy
- business-ready
- modern

### Avoid
- casual slang
- too much technical wording
- oversized text blocks
- repetitive phrases
- vague CTA wording
